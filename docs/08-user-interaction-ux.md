# 08 · 用户交互与 UX(User Interaction & UX)

> 原创整理,基于公开信息与通用 agent 设计模式。代码为示意性伪代码。

Agent 的内核再好,用起来卡手也没人用。这一章讲人与 agent 之间的交互设计——它直接决定 agent 好不好用、用户信不信任它。

## 流式渲染(Streaming UI)

呼应 [06 · 模型交互层](06-model-interaction.md)的流式输出,前端要把增量事件实时渲染出来:

- **文本边到边显示**,降低感知延迟。
- **工具调用可视化**:正在调哪个工具、参数是什么、结果如何,对用户透明。
- **思考/进度提示**:让用户知道 agent 在干活,而不是卡死了。

透明度本身就是信任的来源:用户能看到 agent 在做什么,才敢把更大的权限交给它。

## 中断处理(Interruption)

用户必须能**随时叫停**——这是基本的掌控感:

- 生成中途可打断(Esc / Ctrl-C)。
- 工具执行中途可打断(尤其长命令)。
- 打断后**优雅收尾**:保存当前状态(见 [07](07-session-lifecycle.md))、清理半成品、回到可接受新输入的状态。

```python
def run_turn(state):
    try:
        for event in stream_turn(state):
            if interrupt_requested():
                raise UserInterrupt()
            render(event)
    except UserInterrupt:
        checkpoint(state)        # 存档,便于之后 resume
        cleanup_partial_work()   # 清理半成品
        return AWAIT_USER_INPUT
```

## 确认流(Confirmation Flow)

这是把 [04 · 权限与安全](04-permissions-safety.md)落到 UX 的地方。高风险操作执行前要让用户确认,且确认体验要好:

- **说清三件事**:这个动作要做什么、有什么风险、是否可逆。
- **给出选项**:批准 / 拒绝 / 修改 / "本会话内总是允许此类"。
- **diff 预览**:改文件前展示 diff,让用户看清改了什么再点确认。
- **确认必须来自真实用户**:不接受来自工具结果/被操作内容里的"已授权"声明(injection 防御,见 [04](04-permissions-safety.md))。

记住与 denial tracking 的联动:用户多次拒绝后,agent 应自动转保守(见 [04](04-permissions-safety.md))。

## Plan 模式(先规划后执行)

对复杂/多文件改动,先**只读探索 + 给方案**,经用户批准再动手:

```
plan 模式:只读工具可用,写工具禁用 → 产出方案 → 用户批准 → 切换到执行模式
```

价值:在写代码前对齐意图,避免"跑偏了才发现"。这也呼应 [02](02-tool-design.md) 的模式过滤(plan 模式下工具池禁写)。

## 进度叙述(Narration)——但要克制

agent 该不该一直"汇报"?原则是**克制**:

- **默认安静**:例行的工具调用不需要"我现在要……""让我看看……"这类旁白。
- **值得说才说**:有发现、改方向、遇阻塞时,一句话点明即可。
- **收尾简洁**:任务完成给一两句结论,别把每个文件每个测试都复述一遍——用户一直在看。

过度叙述既费 token 又干扰用户。

## Slash 命令 / 快捷指令

给常用操作提供显式入口(`/help`、`/clear`、`/plan`、自定义命令等):

- 让用户能**直接触发**确定性行为,不必"请求模型去做"。
- 自定义命令可承载项目特定工作流。
- 与可扩展性机制(见 [09](09-extensibility.md))结合,可做成可插拔的命令。

## 多入口一致性

呼应 [01](01-agent-loop.md):CLI / IDE / SDK 共用一套 loop,但**交互表现要适配各自载体**:

- CLI:文本流 + 终端确认提示。
- IDE:内联 diff、侧边栏、点击确认。
- 核心 loop 不变,**渲染与确认的呈现层各自适配**。

## 设计要点清单

- [ ] 流式渲染文本与工具调用,保持过程透明。
- [ ] 支持随时中断;中断后存档、清理、优雅收尾。
- [ ] 确认流说清"做什么/风险/可逆性",改文件给 diff 预览。
- [ ] 确认只认真实用户,拒绝外部内容里的"已授权"。
- [ ] 提供 plan 模式:先只读规划,批准后再执行。
- [ ] 进度叙述要克制:默认安静,值得说才说,收尾简洁。
- [ ] 提供 slash/快捷命令触发确定性行为。
- [ ] 一套 loop,多入口适配各自的渲染与确认呈现。
