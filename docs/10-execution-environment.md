# 10 · 执行环境与沙箱(Execution Environment & Sandboxing)

> 原创整理,基于公开信息与通用 agent 设计模式。代码为示意性伪代码。

[04 · 权限与安全](04-permissions-safety.md)讲的是"要不要让这个动作发生"(策略层);这一章讲"动作发生时被关在多大的笼子里"(机制层)。两者是策略与执行的关系——权限决定意图,沙箱限制后果。

## 为什么需要沙箱

agent 会跑命令、读写文件、可能访问网络。即使有权限确认,也应假设**有些动作会出错或被诱导**(prompt injection),因此要用环境边界把**潜在影响范围(blast radius)**框住。原则:**纵深防御**——权限层之外再加一道机制层。

## 文件系统边界

限制 agent 能读能写的范围:

```
读:可放宽(项目内 + 必要的系统只读路径),但避开敏感文件
写:严格收窄到工作目录及白名单路径
```

- **写白名单**:默认只允许写项目目录、临时目录;项目外的写入要显式放行。
- **敏感文件读取**:对 `.env`、私钥、凭据库等格外谨慎;**即使必须读,也不要把秘密值回显**到响应/日志里(按 key 名引用,而非值)。
- **路径校验**:防目录穿越(`../`)、符号链接逃逸。把用户/模型给的路径解析成绝对路径后再对照白名单。

## 网络边界

- **默认收紧**:除非任务需要,限制出站网络。
- **白名单主机**:只允许访问明确需要的域名/主机。
- **防数据外泄**:不要因为工具结果/网页里的指令,就把代码、秘密、用户数据发往第三方端点(injection + 隐私防御,见 [04](04-permissions-safety.md))。
- URL 里不要带敏感信息(会进日志、referrer)。

## 命令执行隔离(Bash 工具的笼子)

`Bash` 是能力最强也最危险的工具,要重点关:

- **在受控环境里执行**:限定工作目录、限定可写路径、限定网络。
- **超时**:每条命令设超时,防止挂死。
- **资源限制**:必要时限制 CPU/内存/输出大小。
- **输入构造防注入**:命令含用户/模型提供的值时,做正确的引用转义,或用数组式参数化执行,**避免字符串拼接造成命令注入**。
- **子命令绕过的警惕**:像 `git` 这种子命令繁多的命令,静态分析容易被绕过(见 [04](04-permissions-safety.md) 已知弱点),要么用更细的策略,要么收紧环境边界兜底。

```python
# 反例:字符串拼接,易被注入
run(f"grep {user_pattern} file.txt")

# 正例:参数化,避免 shell 解释
run(["grep", user_pattern, "file.txt"])
```

## 工作目录模型(Working Directory)

agent 应有清晰的"当前工作目录"概念:

- 操作默认相对工作目录解析。
- 工作目录纳入会话状态(见 [07](07-session-lifecycle.md)),恢复时重建。
- 切换目录是显式动作,边界随之调整。

## 隔离强度的层次

按需要选择隔离强度(越强越安全也越重):

| 强度 | 手段 | 适用 |
|---|---|---|
| 弱 | 进程内路径/参数校验 | 受信任的本地开发 |
| 中 | 受限子进程(限工作目录/网络/超时) | 一般场景 |
| 强 | 容器/VM/专用沙箱 | 不受信任输入、自动模式、生产 |

自动化程度越高、输入越不可信(见 [04](04-permissions-safety.md) 的信任谱),越应往强隔离走。

## 临时文件与清理

- 临时产物写到专用临时目录,**用完即清**。
- 大工具输出落盘(见 [03](03-context-management.md))也归在受控目录,纳入清理与隐私策略。
- 验证/测试产生的临时文件,任务结束要清理。

## 设计要点清单

- [ ] 把沙箱当作权限之外的第二道防线(纵深防御)。
- [ ] 文件系统:写白名单收窄;敏感文件谨慎且不回显秘密值。
- [ ] 路径解析为绝对路径后校验,防穿越/符号链接逃逸。
- [ ] 网络默认收紧 + 主机白名单;防数据外泄;URL 不带敏感信息。
- [ ] Bash 在受控环境跑:限目录/网络、设超时与资源上限。
- [ ] 命令构造用参数化,杜绝字符串拼接注入。
- [ ] 警惕子命令繁多的命令绕过策略,用环境边界兜底。
- [ ] 清晰的工作目录模型,纳入会话状态。
- [ ] 按输入可信度与自动化程度选择隔离强度。
- [ ] 临时文件用完即清,纳入隐私策略。
